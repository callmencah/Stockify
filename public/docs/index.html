ℹ️ Tentang Stockify

Sistem manajemen inventori modern berbasis web

Stockify adalah aplikasi manajemen inventori berbasis web yang memungkinkan bisnis Anda untuk memantau stok secara real-time, mengelola beberapa gudang sekaligus, dan mengintegrasikan alur pembelian serta penjualan dalam satu platform.

📊

Real-time Tracking

Stok diperbarui secara instan setiap kali ada transaksi masuk atau keluar.

🏪

Multi Gudang

Kelola stok di beberapa gudang dan toko secara bersamaan dari satu dashboard.

📱

Mobile Friendly

Akses penuh dari smartphone. Tampilan responsif untuk semua ukuran layar.

🔒

Role-based Access

Kontrol akses berbasis peran: Admin, Manager, Staff, dan Viewer.

📈

Laporan Lengkap

Nilai stok, riwayat transaksi, penjualan, dan audit trail lengkap.

🔔

Alert Otomatis

Notifikasi otomatis saat stok mendekati reorder point yang ditentukan.

💡
Tech Stack: Next.js 16 · TypeScript · Prisma ORM · SQLite/PostgreSQL · NextAuth.js · Tailwind CSS · TanStack Query · Recharts

⚡ Mulai Cepat

Jalankan Stockify dalam hitungan menit

⚠️
Pastikan Node.js 18+ sudah terinstall di komputer Anda sebelum memulai.
1

Install Dependencies

Buka terminal di folder project Stockify, lalu jalankan: 

npm install
2

Setup Database

Buat dan sinkronkan skema database:

npm run db:push
3

Isi Data Demo & Skala Besar

Masukkan data contoh (barang, gudang, supplier, user). Tersedia 3 pilihan volume: 

npm run db:seed       # Data demo (50 item)
npm run db:seed:10k   # Volume besar (10.000 item)
npm run db:seed:100k  # Volume besar (100.000 item)
4

Jalankan Server

Start development server dan biarkan terminal tetap terbuka: 

npm run dev

Tunggu sampai muncul: ✓ Ready in ~500ms

5

Buka di Browser

Akses aplikasi di: http://localhost:3000

Anda akan diarahkan ke halaman login secara otomatis.

Perintah Berguna

Perintah Fungsi
npm run dev Jalankan server development
npm run build Build untuk production
npm run start Jalankan server production
npm run db:push Sinkronkan skema Prisma ke database
npm run db:studio Buka Prisma Studio (GUI database)
npm run db:seed Isi database dengan data demo (50 item)
npm run db:seed:10k Isi database dengan data skala besar (10.000 item)
npm run db:seed:100k Isi database dengan data skala besar (100.000 item)
npm run db:generate Generate ulang Prisma Client

📦 Master Barang

Kelola data produk/barang beserta informasi lengkapnya

Cara Menambah Barang Baru

1

Klik "Tambah Barang"

Tombol biru di pojok kanan atas halaman Barang (Item).

2

Isi Data Wajib

  • SKU — Kode unik produk (cth: ELEC-LAP-001)
  • Nama Barang — Nama lengkap produk
  • Harga Beli — Harga perolehan dari supplier
  • Harga Jual — Harga jual ke customer
3

Isi Data Tambahan (Opsional)

  • Barcode — Kode EAN/UPC untuk scan
  • Kategori — Pengelompokan barang
  • Satuan — pcs, kg, box, liter, dll
  • Min. Stok & Reorder Point — Batas peringatan stok rendah
4

Aktifkan Tracking (Opsional)

  • Tracking Serial Number — Untuk barang elektronik yang perlu dilacak per unit
  • Tracking Batch/Lot — Untuk produk dengan tanggal kadaluarsa (makanan, obat)
5

Klik "Simpan"

Barang akan langsung tersedia di sistem dan bisa ditambahkan ke inventori.

Field Penjelasan

Field Tipe Keterangan
SKU Wajib Stock Keeping Unit — kode unik internal, tidak bisa diubah setelah dibuat
Barcode Opsional Kode EAN-13, UPC, atau QR untuk scanning
Nama Barang Wajib Nama lengkap produk yang tampil di laporan
Deskripsi Opsional Keterangan tambahan produk
Harga Beli Wajib Digunakan untuk menghitung nilai stok
Harga Jual Wajib Auto-fill saat membuat Sales Order
Min. Stok Dianjurkan Level kritis — badge merah jika stok ≤ nilai ini
Reorder Point Dianjurkan Level peringatan — badge kuning dan notifikasi dikirim
Kategori Opsional Pengelompokan untuk filter laporan
Satuan Opsional pcs, box, kg, liter — tampil di semua transaksi
⚠️
SKU dan Barcode harus unik di seluruh sistem. Jika sudah digunakan, sistem akan menolak dengan error 409.

🏪 Inventori & Stok

Pantau level stok per barang per lokasi secara real-time

Halaman Inventori menampilkan stok terkini untuk semua kombinasi barang × lokasi. Setiap transaksi (PO, SO, Transfer, Opname) langsung memperbarui angka di sini.

Indikator Status Stok

Status Kondisi Warna Progress Bar Tindakan yang Disarankan
Normal Stok > Reorder Point 🟢 Hijau —
Rendah Min Stok < Stok ≤ Reorder Point 🟡 Kuning Buat Purchase Order segera
Kritis 0 < Stok ≤ Min Stok 🔴 Merah Reorder darurat / transfer dari gudang lain
Habis Stok = 0 🔴 Merah Segera restock

Filter & Pencarian

  • Search — Cari berdasarkan nama barang atau SKU
  • Filter Lokasi — Tampilkan stok hanya di gudang/toko tertentu
  • Filter Status — Normal, Stok Rendah, Habis, atau Semua

🔄 Transfer Stok

Pindahkan barang antar gudang atau toko

1

Pilih Lokasi Asal

Gudang atau toko sumber barang yang akan dipindahkan.

2

Pilih Lokasi Tujuan

Sistem otomatis menyembunyikan lokasi yang sama dengan asal.

3

Cari & Pilih Barang

Gunakan kolom pencarian untuk menemukan barang dengan cepat.

4

Masukkan Jumlah

Sistem akan validasi apakah stok di lokasi asal mencukupi.

5

Klik "Transfer Sekarang"

Stok di kedua lokasi langsung diperbarui secara atomik (tidak bisa setengah-setengah).

ℹ️
Transfer menghasilkan 2 transaksi: TRANSFER_OUT dari lokasi asal dan TRANSFER_IN ke lokasi tujuan, keduanya dengan nomor referensi yang sama (cth: TRF-1718234567890).

🚚 Purchase Order

Kelola pemesanan barang ke supplier dan penerimaan barang

Alur Purchase Order

DRAFT Buat PO

Pilih supplier, lokasi tujuan, tambahkan item dengan jumlah dan harga beli. PO tersimpan sebagai Draft.

SENT PO Dikirim ke Supplier

Ubah status ke "Dikirim" setelah PO dikirimkan ke supplier secara manual/email.

PARTIAL Sebagian Diterima

Jika barang datang tidak sekaligus, sistem mencatat penerimaan sebagian.

RECEIVED Selesai Diterima

Semua item sudah diterima. Stok otomatis bertambah di gudang tujuan. Transaksi PURCHASE tercatat.

Cara Menerima Barang

1

Klik ikon ✅ pada baris PO

Hanya muncul untuk PO berstatus Draft, Dikirim, atau Sebagian Diterima.

2

Isi Jumlah Barang yang Diterima

Bisa berbeda dari jumlah order (penerimaan parsial). Sistem pre-fill dengan sisa yang belum diterima.

3

Klik "Konfirmasi Penerimaan"

Stok langsung bertambah. Jika semua item sudah diterima, status PO otomatis menjadi RECEIVED.

✅
Setelah penerimaan, sistem otomatis memeriksa apakah ada barang yang stoknya rendah dan mengirim notifikasi ke Admin & Manager.

🛒 Sales Order

Kelola penjualan dengan pengurangan stok otomatis

⚠️
Stok langsung dikurangi saat Sales Order berhasil dibuat (status CONFIRMED). Pastikan stok mencukupi sebelum membuat SO.

Cara Membuat Sales Order

1

Pilih Customer (Opsional)

SO bisa dibuat tanpa customer untuk penjualan tunai/walk-in.

2

Pilih Lokasi

Stok akan dikurangi dari lokasi yang dipilih.

3

Tambahkan Item

Cari dan pilih barang. Harga jual otomatis terisi dari master data. Anda bisa mengubahnya.

4

Atur Diskon & Pajak

Diskon dan pajak dalam persen (%). Total dihitung otomatis.

5

Klik "Buat Sales Order"

Sistem memvalidasi stok. Jika mencukupi, SO dibuat dan stok langsung dikurangi.

Status Sales Order

Status Keterangan Stok
DRAFT SO dibuat tapi belum dikonfirmasi Belum dikurangi
CONFIRMED SO dikonfirmasi — default saat dibuat ✅ Dikurangi
SHIPPED Barang sudah dikirim ke customer ✅ Dikurangi
DELIVERED Barang sudah diterima customer ✅ Dikurangi
CANCELLED SO dibatalkan ⚠️ Perlu adjustment manual

📋 Stock Opname

Hitung fisik stok dan sesuaikan dengan data sistem

Stock Opname adalah proses penghitungan fisik stok untuk memverifikasi keakuratan data sistem. Setiap selisih akan dicatat dan bisa disesuaikan secara otomatis.

Alur Stock Opname

1

Mulai Opname Baru

Klik "Mulai Opname", pilih lokasi. Sistem otomatis membuat daftar semua barang di lokasi tersebut beserta stok sistem saat ini.

2

Hitung Stok Fisik

Klik tombol "Input" pada baris opname. Masukkan jumlah fisik untuk setiap barang. Selisih dihitung otomatis (+ berarti lebih, - berarti kurang).

3

Simpan Hitungan (Sementara)

Klik "Simpan Hitungan" untuk menyimpan progress tanpa mengubah stok. Bisa dilanjutkan nanti.

4

Selesaikan & Sesuaikan Stok

Klik "Selesaikan & Sesuaikan Stok". Stok di sistem akan diubah sesuai hitungan fisik. Semua penyesuaian tercatat sebagai transaksi OPNAME.

🚨
Perhatian: Setelah klik "Selesaikan", stok sistem akan disesuaikan secara permanen. Pastikan semua hitungan fisik sudah benar sebelum menyelesaikan opname.

📈 Laporan & Analitik

Lima jenis laporan untuk analisis mendalam

💰

Nilai Stok

Nilai inventori per barang per lokasi. Termasuk grafik bar 10 item terbesar. Total nilai keseluruhan ditampilkan di header.

⚠️

Stok Rendah

Semua item yang stoknya ≤ reorder point, diurutkan dari yang paling kritis. Bisa filter per lokasi.

🔄

Riwayat Transaksi

Semua pergerakan stok dengan filter tanggal dan lokasi. Termasuk tipe, referensi, dan user yang melakukan.

📊

Rangkuman Penjualan

Total qty terjual dan pendapatan per barang. Grafik bar 10 best seller. Filter periode dan lokasi.

🕵️

Audit Trail

Log semua aktivitas pengguna: CREATE, UPDATE, DELETE, LOGIN. Hanya bisa diakses Admin.

📅
Semua laporan mendukung filter tanggal (periode) dan filter lokasi. Gunakan kombinasi keduanya untuk analisis yang lebih spesifik.

Export Laporan (PDF & Excel)

Setiap laporan dapat diunduh atau diekspor ke dalam format PDF dan Excel (.xlsx) untuk keperluan arsip atau analisis lebih lanjut di luar sistem.

1

Buka Laporan

Pilih jenis laporan yang ingin Anda lihat dari menu navigasi Laporan.

2

Terapkan Filter (Opsional)

Sesuaikan filter tanggal dan lokasi jika diperlukan untuk menampilkan data yang lebih spesifik.

3

Klik Tombol Export

Pada bagian atas tabel atau halaman laporan, klik tombol Export PDF untuk menyimpan dokumen siap cetak, atau Export Excel untuk mengunduh raw data dalam format spreadsheet (bisa diolah kembali).

Kelola semua titik inventori dalam jaringan distribusi Anda

Tipe Lokasi

Tipe Ikon Kegunaan
WAREHOUSE 🏭 Gudang penyimpanan utama
STORE 🏪 Toko atau outlet penjualan
SUPPLIER 🚚 Lokasi referensi supplier (jarang digunakan)

💡

🏭 Supplier

Database pemasok barang untuk Purchase Order

Data supplier digunakan saat membuat Purchase Order. Field yang tersedia: Kode, Nama, Kontak PIC, Email, Telepon, dan Alamat.

💡
Kode supplier harus unik (cth: SUP-001). PO bisa dibuat tanpa memilih supplier (untuk pembelian tanpa vendor terdaftar).

👤 Customer

Database pelanggan untuk Sales Order

Data customer digunakan saat membuat Sales Order. Field yang tersedia: Kode, Nama, Kontak PIC, Email, Telepon, dan Alamat.

💡
SO bisa dibuat tanpa customer terdaftar (untuk penjualan walk-in/tunai). Customer dipilih secara opsional.

🔌 REST API Reference

Semua endpoint memerlukan sesi autentikasi yang valid

🔒
Semua endpoint dilindungi. Autentikasi dilakukan via session cookie dari NextAuth. Pastikan sudah login sebelum memanggil API.

Barang (Items)

Method Endpoint Deskripsi
GET /api/items Daftar barang (query: search, categoryId, page, limit)
POST /api/items Tambah barang baru
GET /api/items/[id] Detail barang + stok per lokasi
PUT /api/items/[id] Update barang
DELETE /api/items/[id] Nonaktifkan barang (soft delete)

Inventori

Method Endpoint Deskripsi
GET /api/inventory Stok per barang per lokasi (query: locationId, itemId, lowStock)
POST /api/inventory/transfer Transfer stok antar lokasi
GET /api/inventory/transactions Riwayat transaksi stok (query: itemId, locationId, type, page)
GET /api/scan?barcode=xxx Lookup barang berdasarkan barcode atau SKU

Purchase & Sales Order

Method Endpoint Deskripsi
GET /api/purchase-orders Daftar PO (query: status, supplierId)
POST /api/purchase-orders Buat PO baru
GET /api/purchase-orders/[id] Detail PO lengkap
PUT /api/purchase-orders/[id] Update status/catatan PO
POST /api/purchase-orders/[id]/receive Terima barang dari PO
GET /api/sales-orders Daftar SO (query: status, customerId)
POST /api/sales-orders Buat SO baru (stok langsung dikurangi)
GET /api/sales-orders/[id] Detail SO lengkap

Laporan

Method Endpoint Query type
GET /api/reports stock_value · low_stock · transactions · sales_summary · purchase_summary · audit_log

Contoh Request

# Cari barang berdasarkan barcode
GET /api/scan?barcode=8901234567890

# Daftar stok rendah di gudang tertentu
GET /api/inventory?locationId=clxxx&lowStock=true

# Laporan nilai stok periode tertentu
GET /api/reports?type=transactions&startDate=2024-01-01&endDate=2024-12-31

🔔 Notifikasi

Sistem peringatan otomatis untuk kondisi penting

Tipe Trigger Penerima
LOW_STOCK Stok barang ≤ reorder point setelah penerimaan PO Admin, Manager
PO_RECEIVED Semua item PO telah diterima (status RECEIVED) Admin
SO_CONFIRMED Sales Order berhasil dikonfirmasi Admin, Manager

Notifikasi diperbarui setiap 30 detik. Badge merah di ikon lonceng menunjukkan jumlah notifikasi yang belum dibaca.

🕵️ Audit Trail

Rekam jejak semua aktivitas pengguna di sistem

Setiap aksi penting dicatat secara otomatis ke dalam Audit Log, termasuk siapa yang melakukan, kapan, dan apa yang berubah.

Aksi Contoh
LOGIN User berhasil login ke sistem
CREATE Menambah barang baru, membuat PO/SO
UPDATE Mengedit data barang, mengubah status PO

🔍
Audit Log dapat diakses melalui menu Laporan → tab Audit Trail. Hanya Admin yang bisa melihat log ini.

❓ FAQ — Pertanyaan Umum

Jawaban untuk masalah yang sering ditemui

❓ Bagaimana cara menambah stok awal barang baru?

Setelah barang dibuat di Master Barang, buat Purchase Order dengan supplier "Stok Awal" lalu terima barangnya. Stok akan otomatis masuk ke gudang yang dipilih. Alternatif: gunakan Stock Opname — masukkan barang baru dengan stok fisik yang diinginkan.

❓ Apakah bisa undo/membatalkan transaksi yang sudah terlanjur?

Tidak ada fitur undo otomatis. Untuk koreksi: (1) Jika PO salah, ubah status ke CANCELLED. (2) Jika SO salah, ubah ke CANCELLED lalu lakukan adjustment stok manual via Stock Opname. (3) Untuk transfer yang salah, lakukan transfer balik.

❓ Bagaimana cara scan barcode di HP?

Buka aplikasi di browser HP, gunakan fitur pencarian di halaman Barang atau Inventori. Ketik barcode secara manual atau hubungkan scanner Bluetooth ke HP. Fitur kamera scan via @zxing/library sudah terinstall dan bisa diintegrasikan.

❓ Database apa yang disarankan untuk production?

Untuk production, gunakan PostgreSQL. Ubah DATABASE_URL di .env.local dan ganti provider di prisma/schema.prisma dari sqlite ke postgresql, lalu jalankan npm run db:push.

❓ Bagaimana cara backup database?

Untuk SQLite, cukup copy file prisma/dev.db. Untuk PostgreSQL, gunakan pg_dump. Disarankan setup backup otomatis harian menggunakan cron job atau layanan managed database (Supabase, PlanetScale, Railway).

❓ Mengapa NEXTAUTH_SECRET penting?

NEXTAUTH_SECRET digunakan untuk menandatangani JWT token sesi. Jika diubah, semua sesi yang ada akan invalid dan semua user harus login ulang. Gunakan string acak yang panjang dan jangan pernah expose ke publik.

❓ Apakah bisa deploy ke Vercel?

Ya! Vercel adalah platform yang direkomendasikan. Set environment variables: DATABASE_URL, NEXTAUTH_SECRET, NEXTAUTH_URL. Gunakan PostgreSQL (Supabase gratis atau Vercel Postgres) karena SQLite tidak persistent di serverless.

💡 Tips & Trik

Best practice penggunaan Stockify

📐

Konsisten dalam SKU

Gunakan format yang terstruktur: [KATEGORI]-[SUBKAT]-[NOMOR], cth: ELEC-LAP-001. Ini memudahkan pencarian dan pelaporan.

⚠️

Atur Reorder Point

Selalu isi Reorder Point dan Min. Stok untuk setiap barang. Ini adalah kunci agar alert otomatis berfungsi dengan tepat.

📅

Opname Rutin

Lakukan stock opname minimal 1x per bulan atau per gudang. Ini menjaga akurasi data dan mendeteksi selisih lebih awal.

🔐

Role yang Tepat

Jangan berikan role ADMIN sembarangan. Buat akun terpisah untuk setiap karyawan agar audit trail akurat.

🗃️

Gunakan Kategori

Buat hierarki kategori yang jelas sebelum mulai input barang. Ini sangat membantu saat filtering laporan.

🔢

Barcode Konsisten

Pastikan setiap barcode hanya untuk 1 barang. Gunakan format EAN-13 untuk kompatibilitas dengan scanner fisik.
